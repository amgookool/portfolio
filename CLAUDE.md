## Architecture

This is a **TanStack Start** SSR portfolio site — React 19, TanStack Router (file-based), Nitro as the server adapter, Tailwind CSS v4, and Vite.

**Entry points**

- `vite.config.ts` — wires together the Nitro, TanStack Start, Tailwind, and React Vite plugins
- `src/routes/__root.tsx` — root shell: injects global CSS, `<Header>`, `<Footer>`, and the TanStack devtools panel. Also contains the inline theme-init script that runs before hydration to prevent flash.
- `src/routeTree.gen.ts` — auto-generated by `tsr generate`; never edit manually

**Routing**

Routes live in `src/routes/` and follow TanStack file-based conventions. Adding a file auto-registers the route; run `pnpm generate-routes` to sync the generated tree. The root layout (`__root.tsx`) uses `shellComponent` rather than a standard component, which is the TanStack Start SSR pattern.

**Theme system**

Three-way toggle: `light` / `dark` / `auto`. Preference is stored in `localStorage` under the key `theme`. On the server-rendered shell, an inline `<script>` in `__root.tsx` reads that value and applies `data-theme` / class to `<html>` before React hydrates, eliminating flash. `ThemeToggle.tsx` cycles through the three modes. Dark-mode CSS variables live in `src/styles.css` under `[data-theme="dark"]` and `@media (prefers-color-scheme: dark) :root:not([data-theme="light"])`.

**Styling conventions**

Tailwind v4 is used for layout and spacing. Semantic design tokens are CSS custom properties defined in `src/styles.css` (e.g. `--lagoon`, `--sea-ink`, `--surface`). Reusable utility classes like `.island-shell`, `.display-title`, `.island-kicker`, `.nav-link`, `.demo-button`, `.field`, `.input`, `.textarea` are also defined there. Use these classes rather than duplicating their Tailwind equivalents inline.

**Package manager:** `pnpm` (workspace config in `pnpm-workspace.yaml`). Do not use npm or yarn.

**Path aliases:** `#/*` maps to `src/*` (configured in `package.json` `imports` field and `tsconfig.json`).
